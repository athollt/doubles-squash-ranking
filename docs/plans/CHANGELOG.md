## Step 01 — Project scaffold & dev environment

**Date**: 2026-06-05

### Delivered

- Next.js 16.2.7 (App Router, TypeScript strict, no `src/` dir) — spec said "15"; 16 is the current latest from `create-next-app`. No functional difference for this project.
- Tailwind CSS v4 via `@tailwindcss/postcss` — v4 uses `@import "tailwindcss"` in globals.css instead of v3 directives.
- Prisma 7 initialised with PostgreSQL provider, empty schema. Prisma 7 generates to `app/generated/prisma` and uses a `prisma.config.ts` (new in v7 vs v5/v6).
- Docker Compose: `postgres:16-alpine` on port 5433.
- Vitest 4.x configured with jsdom + React Testing Library + jest-dom setup. `npm run test` uses `--passWithNoTests` so empty suite exits cleanly.
- Playwright configured with Chromium project, `e2e/` dir, dev server auto-start.
- ESLint v9 flat config (Next.js recommended) — generated by `create-next-app`.
- `.env.example` with all required env vars (no secrets).
- README updated with purpose, local setup, test, and build instructions.
- `.gitignore` extended: postgres data volume, Playwright output dirs.
- `next.config.ts` with `output: "standalone"`.
- shadcn/ui base config initialised (style: base-nova, Tailwind v4, `components/ui/button.tsx` + `lib/utils.ts` generated).

### Deviations from spec

- Next.js version is 16.2.7, not 15. `create-next-app` installs the latest release; 16 is fully compatible.
- Prisma 7 uses `prisma-client` generator (not `prisma-client-js`) and a `prisma.config.ts` config file. Functionally equivalent to the spec's intent.
- Tailwind CSS v4 syntax differs from v3 (no `tailwind.config.js`, no `@tailwind` directives). shadcn/ui handles this correctly.

### Validation

- `npm run build` — ✅ zero errors
- `npm run test` — ✅ passes (no test files, exits cleanly with `--passWithNoTests`)
- `docker compose up -d postgres` — ✅ Postgres reachable on port 5433
- `npx prisma db push` — ✅ connected successfully, schema in sync
- `docker compose down` — ✅ clean teardown
